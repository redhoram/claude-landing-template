# Conversion Copy

Core belief: visitors skim, doubt, and leave. Copy earns each next second of attention
by being **specific about an outcome the reader already wants**. Clever is optional;
clear is mandatory; honest is non-negotiable.

## 1. Headlines — formulas, then the specificity test

Reach for one of these shapes (≤8 words where possible):

- **Outcome-first**: the after-state, not the tool. "Ship features, not fires."
- **How-without-pain**: "X without Y" — "Hire faster without the resume pile."
- **Named pain as question**: "Still copy-pasting candidate data at 11pm?"
- **Proof-led**: a real number up front. "From 3 days to 20 minutes per report."
- **Category + twist** (for hero only): what it is + why it's different, in one line.

Then the **specificity test**: could a competitor paste this headline onto their site
unchanged? If yes, sharpen until it could only be about THIS product. "Work smarter"
fails; "Screen 200 CVs before your coffee cools" passes.

## 2. Choosing the arc — PAS vs AIDA (one page, one arc)

- **PAS** (Problem → Agitate → Solve): for pain-aware visitors — they know the problem,
  don't know you. Cold traffic, new categories. Agitate = make the cost of the problem
  vivid and concrete (time, money, missed chances), never invented.
- **AIDA** (Attention → Interest → Desire → Action): for product-aware visitors —
  comparing options. Lead with the strongest differentiator, build desire with proof.
- Pick ONE per page based on the audience in `.craft/copy.md`; record the choice.
  A page that agitates AND demos AND lists 12 features follows no arc — it rambles.

## 3. Proof hierarchy — strongest available, honestly

Order of persuasive power; always use the highest tier you HONESTLY have:

1. **Real numbers from the product** ("cut screening time 70%") — only if true and sourced.
2. **Named testimonial** with role/company — real people, real quotes.
3. **Logos / usage counts** — real customers, real counts.
4. **Concrete capability statements** — "reads PDF, DOCX, and scanned CVs" beats "powerful parsing".
5. Nothing yet? Say what the product DOES, precisely. Precision is itself credibility.

Never fabricate any tier. Placeholder proof is marked `[ILLUSTRATIVE]` and the user is
told to replace it before launch. A fake testimonial converts once and destroys trust forever.

## 4. Objection preemption — answer doubts before the CTA

Every landing page fights the same four doubts; place answers along the arc, not in a heap:

- **"Too expensive"** → anchor against what it replaces (hours, tools, hires).
- **"Too much effort"** → time-to-value: "first result in 10 minutes", "no setup".
- **"Won't work for my case"** → name the specific use cases/segments it serves.
- **"Can I trust this?"** → guarantee, free tier, data handling, real contact.

FAQ section = structured objection dump: questions phrased as users actually ask them,
one honest answer each. Strongest objection first, second-strongest last (serial position).

## 5. CTA & microcopy

- Primary CTA: **verb-first + outcome-flavored** — "Start screening free",
  "Get the checklist". Never "Submit", "Learn more", "Click here" as primary.
- One primary CTA per page, repeated where the arc peaks (hero, after proof, end).
  Secondary CTA (if any) is visually quiet and never adjacent to the primary.
- **Friction reducers** live next to the button, not in the footer: "No credit card",
  "2-minute setup", "Cancel anytime" — only the TRUE ones.
- Form microcopy: label above field, example inside (`you@company.com`), error text
  says the fix ("Use your work email"), button continues the sentence the visitor
  is thinking ("Send me the report").

## 6. Language rules

- **Banned openers/fillers**: "Welcome to", "Elevate your", "Unleash", "Empower",
  "Revolutionize", "Take X to the next level", "In today's fast-paced world",
  "seamless"/"cutting-edge"/"innovative" as empty adjectives.
- "You" outnumbers "we" — count it. The reader is the hero; the product is the tool.
- Active voice, present tense, short sentences. If a sentence works without a word, cut it.
- Concrete noun beats abstract noun: "spreadsheet" beats "workflow solution".
- Write for skimming: headline + bolded fragments must tell the whole story alone.

## Done check

- [ ] Headline passes the specificity test (competitor couldn't reuse it)
- [ ] ONE arc chosen (PAS or AIDA) and recorded; section flow follows it
- [ ] Every claim: highest honest proof tier, zero fabrication, `[ILLUSTRATIVE]` marked
- [ ] All four standard objections answered somewhere along the arc
- [ ] Primary CTA verb-first, repeated at arc peaks, friction reducers adjacent
- [ ] Zero banned phrases; "you" > "we"; skim-only read still tells the story
